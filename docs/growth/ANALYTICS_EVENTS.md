# Analytics Events

## Principle
Track explicit product milestones, not manipulative compulsion metrics.

## Core Events
- `landing_viewed`
- `waitlist_form_started`
- `waitlist_submitted`
- `onboarding_started`
- `onboarding_submitted`
- `profile_saved`
- `matches_viewed`
- `match_candidate_opened`
- `conversation_created`
- `message_sent`
- `report_submitted`
- `block_applied`

## Properties To Consider
- city
- neighborhood
- language set
- relationship intent
- neurotype preference mode
- acquisition source

## Metrics To Avoid
- streaks
- time-on-app as a vanity KPI
- pressure-based nudge open rates
